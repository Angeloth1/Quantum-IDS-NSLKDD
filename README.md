## Περιγραφή & Στόχος
Ο κύριος στόχος είναι η διερεύνηση της αποδοτικότητας των κβαντικών αλγορίθμων στην ταξινόμηση κακόβουλης κίνησης. Η έρευνα εστιάζει στην **"Κβαντική Οικονομία"**: πώς μπορούμε να συμπιέσουμε την πληροφορία ενός μεγάλου δικτυακού dataset ώστε να τρέξει σε NISQ (Noisy Intermediate-Scale Quantum) υπολογιστές με ελάχιστα Qubits.

## Τρέχουσα Πρόοδος
- [x] **Data Preprocessing:** Καθαρισμός και "έξυπνο" Stratified Split (βάσει Label & Packet Size).
- [x] **Feature Engineering:** Μετάβαση από One-Hot Encoding σε **Target Encoding**.
- [x] **Dimensionality Reduction:** Υλοποίηση PCA με στόχο τις 16 κύριες συνιστώσες (PCA-16).
- [x] **Model Training:** Εκπαίδευση Variational Quantum Classifier (VQC).

## Dataset & Εγκατάσταση
Το project χρησιμοποιεί το **NSL-KDD**, μια βελτιωμένη έκδοση του κλασικού KDD'99.

* **Λήψη Δεδομένων:** Μπορείτε να κατεβάσετε το dataset από το [University of New Brunswick (UNB)](https://www.unb.ca/cic/datasets/nsl.html).
* **Προετοιμασία:** Τοποθετήστε τα αρχεία `.txt` (π.χ. `KDDTrain+.txt`) στον φάκελο `/data`.

### Οδηγίες Χρήσης:
1. Εγκαταστήστε τις βιβλιοθήκες:  
   `pip install -r requirements.txt`
2. Εκτελέστε το κεντρικό αρχείο:  
   `jupyter notebook notebooks/main.ipynb`

## Μεθοδολογία & Πλάνο Υλοποίησης
Η στρατηγική μας βασίζεται σε δύο πυλώνες που συζητήθηκαν και επιλέχθηκαν για τη βελτιστοποίηση του μοντέλου:

1.  **Target Encoding vs One-Hot:** Αντί για την "έκρηξη" διαστάσεων (121 στήλες) του One-Hot, χρησιμοποιήσαμε Target Encoding. Αυτό μείωσε τις στήλες σε 22, ενσωματώνοντας τη στατιστική πιθανότητα επίθεσης απευθείας στην τιμή της μεταβλητής.
2.  **Ο "Χρυσός Κανόνας" των 16 Components:** Εφαρμόσαμε PCA για να καταλήξουμε σε **16 χαρακτηριστικά**. Η επιλογή του 16 ($2^4$) είναι στρατηγική:
    * **Amplitude Encoding:** Επιτρέπει την κωδικοποίηση όλης της πληροφορίας σε μόλις **4 Qubits**.
