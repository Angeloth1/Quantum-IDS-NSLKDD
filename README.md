## 🧠 Μεθοδολογία & Πλάνο Υλοποίησης
Η στρατηγική μας βασίζεται σε δύο πυλώνες που συζητήθηκαν και επιλέχθηκαν για τη βελτιστοποίηση του μοντέλου:

1.  **Target Encoding vs One-Hot:** Αντί για την "έκρηξη" διαστάσεων (121 στήλες) του One-Hot, χρησιμοποιήσαμε Target Encoding. Αυτό μείωσε τις στήλες σε 22, ενσωματώνοντας τη στατιστική πιθανότητα επίθεσης απευθείας στην τιμή της μεταβλητής.
2.  **Ο "Χρυσός Κανόνας" των 16 Components:** Εφαρμόσαμε PCA για να καταλήξουμε σε **16 χαρακτηριστικά**. Η επιλογή του 16 ($2^4$) είναι στρατηγική:
    * **Amplitude Encoding:** Επιτρέπει την κωδικοποίηση όλης της πληροφορίας σε μόλις **4 Qubits**.
    * **Angle Encoding:** Επιτρέπει μια διαχειρίσιμη προσομοίωση **16 Qubits** σε συμβατικό hardware.

## 🔮 Μελλοντικά Βήματα (Roadmap)
* **Σενάριο Α (Amplitude):** Στόχος η μέγιστη πυκνότητα πληροφορίας σε ελάχιστα Qubits.
* **Σενάριο Β (Angle):** Στόχος η ανθεκτικότητα στον θόρυβο (shallow circuits).
* **Benchmark:** Σύγκριση των δύο μεθόδων ως προς το Accuracy και το Training Time.
